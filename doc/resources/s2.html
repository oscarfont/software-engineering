Task 1: Elaboration of two Use Cases.

1. PURCHASE A BOOK

• Use case: Purchase a book.
• Contex: A person wants to purchase a book from our webpage.

• Primary Actors: Client.
• Support Actors: None.

• Preconditions:
- The user can communicate with the system.
- The user has to be able to pay for the book with any of the ways we offer him to do it.
• Successful postconditions:
- The user purchases the book successfully (which means, the book was in stock and the user could have paid for it).
• Unsuccessful postconditions:
- There was not the book in stock.
- There was any kind of error when the user attempted to pay for the book.
• Basic Flow:
1. The user looks for the book he wants.
2. The user adds the book to the cart.
3. The user asks whether he wants a physical copy or a digital one after the system asks him.
4. The system offers the user ways to pay for the book and the user chooses one of them.
5. The user confirms the purchase and pays for it.
• Extensions:
1a. The user does not have enough information to find the book.
1a1. The user has an option to do an advanced search.
1b. The user wants to buy a book that we do not have in the catalogue.
1b1. Go to step one.
2a. The user can choose how many copies he wants to add to the cart.
2a1. Go to step 3.
3a. If the user chooses a physical copy.
3a1. Give the information to the system of where the user wants to pick up the book.

2. VALIDATE UPLOADED WORK

• Use case: Validate uploaded book.
• Contex: The editor wants to validate an uploaded work by an author.

• Primary Actors: Editor.
• Support Actors: None.

• Preconditions:
- The editor can have access to the system.
- The author has to have uploaded the work in the right format.
• Successful postconditions:
- The editor validates the uploaded work, classifies the work and chooses whether to include the work in one of the editorial's collections
• Unsuccessful postconditions:
- The editor could not validate the work because there was an error either in the work's format or the book's layout design.
• Basic Flow:
1. The editor looks for the work to validate.
2. The system opens the work and offers the option to download it.
3. The editor checks the work to decide to validate it or not.
4. The editor decides to validate or not. Then he/she looks for the option to do so.
5. The system asks the editor if he/she is sure he/she wants to validate the work or not.
6. The editor looks for the options to classify the work in one collection.
7. The system offers the editor all the categories to classify the work.
8. The editor decides in which collection to classify the work.
•Extensions:
1a. The editor can not find the book.
1a1. Go to step 1.
2a. The editor downloads the work to check it on his PC.
2a1. Follow with step 3.
3a. The editor finds format errors.
3a1. The editor informs the author.
3b. The editor finds layout errors.
3b1. The editor informs the Layour designer.
7a. The editor can not find a category that fits the book.
7a1. The editor includes the book in the category "Unclassifed".
